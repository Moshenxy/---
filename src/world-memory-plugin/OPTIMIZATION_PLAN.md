# 世界记忆插件 V2 优化计划

## 一. 整体架构理解

在完整阅读了项目的所有代码后，我确认该插件不仅仅是一个上下文注入工具，而是一个设计精巧、功能完备的“人工心智模拟框架”。其核心由**感知输入**、**分层记忆系统**、**反思推理机制**、**新陈代谢（遗忘与清理）**以及一个强大的**可视化内省界面（人格图谱）**组成。

该项目代码质量极高，模块化清晰，遵循了现代 TypeScript/Vue 的最佳实践。因此，我的优化计划将不会聚焦于“修复问题”，而是着眼于“锦上添花”，在现有坚实的基础上提出增强功能和改进建议。

## 二. 优化方向建议

我建议从以下几个方面对插件进行优化和增强：

### A. 功能与逻辑增强 (Feature & Logic Enhancements)

1.  **认知冲突检测 (Cognitive Dissonance)**:
    *   **现状**: 当前系统会不断添加新认知。
    *   **建议**: 在 `PreprocessorHandler` 或 `LifecycleService` 中增加一个新模块，用于检测新生成的认知是否与已有的核心“本性”或重要认知产生冲突。如果检测到冲突，可以：
        *   生成一个特殊的“内心挣扎”或“认知失调”条目，影响AI的短期行为。
        *   在人格图谱中用特殊的颜色或连线高亮显示冲突的节点。

2.  **手动关联与分离 (Manual Association)**:
    *   **现状**: 记忆与认知的关联完全由AI自动生成。
    *   **建议**: 在“人格图谱”的 `NodeDetail` 弹窗中，增加一个“关联管理”功能。允许用户：
        *   手动将一个“认知”或“本性”链接到某个“情景记忆”上。
        *   手动断开一个已有的链接。
        *   这将赋予用户终极的记忆管理权。

3.  **记忆摘要AI优化 (Summary AI Refinement)**:
    *   **现状**: `narrator.txt` 指导主AI生成记忆日志。
    *   **建议**: 引入一个独立的、专门用于“总结”的AI调用。在 `MessageHandler` 中，获取到原始的 `<memory_log>` 后，不直接保存，而是再调用一次专门的“摘要AI”，对 `full_context` 部分进行更精炼、更准确的总结，生成 `summary` 文本。这能提高记忆摘要的质量。

### B. 性能与效率优化 (Performance & Efficiency)

1.  **图谱渲染优化 (Atlas Rendering)**:
    *   **现状**: `AtlasPanel.vue` 在数据变化时会完全重新渲染整个图谱 (`disposeScene` -> `renderGraph`)。
    *   **建议**: 改造 `renderGraph` 函数，使其支持“差量更新”。当仅增加少量节点或时间线变化时，只添加/删除/更新变化的 `three.js` 对象，而不是销毁和重建所有对象，这能极大提升大规模图谱的交互流畅度。

2.  **后台服务节流 (Service Throttling)**:
    *   **现状**: `ChatSynchronizer` 在每次 `CHAT_CHANGED` 时都会触发。
    *   **建议**: 为 `onChatChanged` 方法增加一个 `debounce`（防抖）处理。例如，在用户快速连续删除多条消息时，只在最后一次删除操作的5秒后执行一次同步检查，避免不必要的重复计算。

### C. 用户体验改进 (UX Improvements)

1.  **提供更明确的反馈**: 在“人格创生”、“删除记忆”等长时间操作期间，除了 `toastr` 提示外，可以在UI上显示更明显的加载动画或进度条。
2.  **增强图谱交互**: 在图谱中增加鼠标悬停高亮相关节点的功能，让用户更容易发现节点间的联系。
3.  **时间线改进**: 在时间线滑块上增加关键事件的标记点（例如，有新认知生成的楼层）。

## 三. 已完成的优化与修复 (2026-04-06)

1.  **图谱弹窗 UI 修复**: 修复了 `NodeDetail` 弹窗在内容过多时被截断且无滚动条的问题。通过使用 Flexbox 布局，使弹窗自适应父容器并正确触发内部滚动。
2.  **图谱可视化升级**: 将 `AtlasPanel` 中的 2D 径向力 (`forceRadial`) 升级为 3D 版本，并移除了视觉干扰较大的连线。现在，本性、认知和记忆节点以清晰的 3D 球壳分层环绕中心，极大地提升了图谱的立体感和可读性。
3.  **支撑记忆交互增强**: 修复了支撑记忆列表不显示的问题，并将其升级为可点击的链接。用户现在可以直接点击支撑记忆的 ID，在图谱中快速跳转并查看该记忆的详情。
4.  **记忆细节编辑功能**: 在 `NodeDetail` 中，为情景记忆的“闪光灯碎片”（视觉、听觉、体感）和“关键词”添加了内联编辑功能，并同步更新了后端的 `TavernService` 以支持这些嵌套属性的保存。
