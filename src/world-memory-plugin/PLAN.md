# 世界记忆插件 V2 优化与重构计划

## 1. 需求理解
本项目是一个为SillyTavern酒馆设计的“世界记忆插件”，旨在通过AI驱动的记忆与人格系统，赋予角色动态成长和演变的能力。
核心机制为“分析-注入”模型：
- **事前注入 (PreprocessorHandler)**: 在AI生成回复前，拦截请求，调用反思AI分析近期记忆，提取“认知”和“本性”，并将其作为上下文注入到系统提示词中。
- **事后提取 (MessageHandler)**: 在AI生成回复后，解析回复中的 `<memory_log>` 标签，将其作为“情景记忆”保存。

当前代码已经具备了良好的模块化基础（core, services, ui, templates, types, utils）。
您的需求是：在深入理解现有架构的基础上，进行进一步的优化和重构，遵循TypeScript最佳实践，完善目录结构、模块化划分，并为每个模块补充README文档和详细注释。同时，在正式编码前，先与您沟通理解和计划。

## 2. 优化方向与计划

基于项目中的 `OPTIMIZATION_PLAN.md` 和 TypeScript 最佳实践，我计划分以下几个阶段进行：

### 阶段一：架构梳理与文档完善
- 为 `core`, `services`, `ui`, `types`, `utils` 等核心目录创建 `README.md`，简要说明模块职责。
- 补充和完善关键类、函数、接口的 TSDoc 注释，确保代码自解释性。
- 梳理现有的模块依赖关系，确保没有循环依赖。

### 阶段二：功能与逻辑增强 (A类优化)
- **认知冲突检测**: 在反思阶段 (`PreprocessorHandler` 或 `LifecycleService`) 引入冲突检测机制，当新认知与核心本性冲突时，生成特殊标记或内心挣扎事件。
- **手动关联与分离**: 增强UI，允许用户在“人格图谱”中手动管理记忆、认知、本性之间的关联。
- **记忆摘要AI优化**: 引入独立的摘要AI调用，对冗长的情景记忆进行二次精炼，提高记忆质量。

### 阶段三：性能与效率优化 (B类优化)
- **图谱渲染优化**: 改造 `AtlasPanel.vue` 的渲染逻辑，引入差量更新机制，避免每次数据变动都全量重建3D场景。
- **后台服务节流**: 为 `ChatSynchronizer` 等高频触发的服务添加防抖(debounce)或节流(throttle)机制。

### 阶段四：代码结构与工程化重构
- 检查并优化单例模式的使用，考虑引入简单的依赖注入，降低模块间耦合。
- 确保所有数据结构都有严格的 Zod 校验和 TypeScript 类型定义。
- 统一错误处理和日志记录机制。
